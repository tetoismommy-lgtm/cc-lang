01

What is CC?

CC is a scripting language built specifically for talking to APIs and web services. Where curl is powerful but cryptic, and Python's requests library requires a full project setup, CC gives you HTTP as a first-class citizen — with variables, logic, loops, and functions built right in.

Every line in CC starts with a command — a verb that tells CC exactly what to do. No boilerplate. No semicolons. No class definitions. Just commands.

INVOKE HTTP
INVOKE JSON

FETCH "https://api.github.com" "/zen" INTO response OR FAIL
SPEAK response_status
SPEAK response_body

That's a real HTTP request with error handling in 5 lines. No imports, no setup, no surrounding code needed.

02

Running CC

CC files use the .cc extension. Once installed, run any script from your terminal:

$cc myprogram.cc

This works from any folder on your machine. No Python, no virtual environments, no setup.

03

The REPL

Run cc with no arguments to open the interactive shell. Type commands one at a time. Variables persist between lines — perfect for exploring an API before writing a full script.

$cc
>> INVOKE HTTP
>> FETCH "https://api.github.com" "/zen" INTO response OR FAIL
>> SPEAK response_body
Non-blocking is better than blocking.
>> SHOW

CC saves your command history to ~/.cc_history automatically. On Windows, install pyreadline3 for arrow-key history in the terminal.

Type exit or quit to leave.

💡 Workflow tip Use the REPL to explore an API interactively. Once you have a working sequence of commands, paste them into a .cc file and run it as a script. Your REPL history is your first draft.
04

Syntax Rules

CC has four rules. Every valid CC program follows all of them.

RuleExample
KEYWORDS are UPPERCASESPEAK, GRANT, FETCH
variable names are lowercasename, response, total
"Strings in double quotes""hello", "https://api.example.com"
One statement per lineEach command lives on its own line
⚠ Case sensitive speak is not the same as SPEAK. Keywords must be fully uppercase or CC will not recognize them.
05

SPEAK

Prints a value to the screen. Works with strings, numbers, variables, and math expressions.

SPEAK "Hello world"
SPEAK x
SPEAK x + y
SPEAK response_status
06

GRANT

Creates or updates a variable. The keyword TO is required between the name and the value. Variables can hold numbers, strings, or the result of a math expression.

GRANT x TO 10
GRANT name TO "CC"
GRANT result TO x * 2 + 5
GRANT x TO x + 1
07

IMMORTAL

Creates a constant — a variable that can never be reassigned. Use it for base URLs, API versions, or any value that should never change. Trying to reassign it causes an immediate error.

IMMORTAL base_url TO "https://api.example.com"
IMMORTAL version TO 2
08

ASK

Prompts the user for input and stores the answer in a variable. CC automatically detects whether the response is a number or a string — no casting required.

ASK token "enter your API token"
ASK limit "how many results"
09

SILENCE

Marks the rest of the line as a comment. CC ignores it entirely. Must appear at the start of its own line — not after code on the same line.

SILENCE authenticate and fetch user profile
SILENCE --- setup ---
GRANT base TO "https://api.github.com"
10

BETRAY

Throws a custom error and stops the program immediately. Use it to guard against bad states — a failed request, an invalid response, or an impossible condition.

FETCH base "/user" INTO response OR CONTINUE
OBEY response_status != 200
    BETRAY "authentication failed"
END
11

SHOW

Dumps every variable currently in memory to the screen. Most useful in the REPL after making requests — lets you see what CC is holding without printing individual variables.

SHOW
12

OBEY / ALSO OBEY / OTHERWISE

Conditional execution. If the condition is true, the block runs. Add ALSO OBEY for else-if chains. Add OTHERWISE for a final fallback. Every block ends with END.

Supported comparisons: > < >= <= == != — works on both numbers and strings.

OBEY response_status == 200
    SPEAK "success"
ALSO OBEY response_status == 401
    BETRAY "unauthorized"
ALSO OBEY response_status == 404
    SPEAK "not found"
OTHERWISE
    SPEAK "unexpected response"
END
OBEY response_error == "none"
    SPEAK "request succeeded"
END
13

WHILE

Repeats a block as long as a condition stays true. CC stops automatically after 100,000 iterations to prevent infinite loops.

GRANT page TO 1
WHILE page <= 5
    FETCH base "/items" INTO response OR FAIL
    SPEAK response_body
    GRANT page TO page + 1
END
14

REPEAT

Runs a block an exact number of times. The count can be a literal number or a variable.

REPEAT 3
    FETCH base "/ping" INTO response OR CONTINUE
    SPEAK response_status
END
15

STOP

Breaks out of the current loop immediately. Useful for exiting a WHILE loop early once a condition is met — like when a health check passes.

WHILE ready == 0
    FETCH base "/health" INTO response OR CONTINUE
    OBEY response_status == 200
        GRANT ready TO 1
        STOP
    END
END
16

CONTRACT

Defines a reusable block of code. Call it by writing its name on a line. Contracts share memory with the rest of the program — any variable they change stays changed after the call.

Add WITH to accept arguments. Use RETURN to send a value back — it gets stored in __return__.

GRANT total TO 0

CONTRACT add_ten
    GRANT total TO total + 10
END

add_ten
add_ten
SPEAK total  SILENCE prints 20
CONTRACT double WITH n
    RETURN n * 2
END

double WITH 7
SPEAK __return__  SILENCE prints 14
17

Lists

Ordered collections. Create one with GATHER, add items with ADD, read by index with GET (starting at 0), delete with REMOVE, and get the count with SIZE.

GATHER endpoints
ADD endpoints "/users"
ADD endpoints "/posts"
ADD endpoints "/comments"

SIZE endpoints INTO total
GRANT i TO 0
WHILE i < total
    GET endpoints i INTO path
    FETCH "https://api.example.com" path INTO response OR CONTINUE
    SPEAK response_status
    GRANT i TO i + 1
END
18

String Operations

CC has five string operations. All store their result INTO a variable.

CommandWhat it does
LENGTH str INTO nGet the number of characters in a string
JOIN a b INTO rConcatenate two strings together
SLICE str start end INTO rExtract a portion of a string. Start is inclusive, end is exclusive.
ORD char INTO nGet the ASCII number for a character. e.g. "A" → 65
CHR n INTO charConvert an ASCII number back to a character. e.g. 65 → "A"
GRANT url TO "https://api.example.com"
LENGTH url INTO n          SILENCE n = 23
SLICE url 8 11 INTO part   SILENCE part = "api"
JOIN url "/users" INTO full SILENCE full = "https://api.example.com/users"
💡 SLICE with variable indices SLICE supports simple math in the index positions: SLICE body i i + 1 INTO ch lets you step through a string character by character.
19

Files

Write API responses to disk, or read config files and saved data back into CC.

SILENCE save a response to disk
FETCH "https://api.example.com" "/data" INTO response OR FAIL
WRITE "response.json" response_body

SILENCE read it back later
READ "response.json" INTO contents
SPEAK contents
⚠ Note WRITE overwrites files without warning. READ on a file that doesn't exist causes an error.
20

ENV / DOTENV

Read API keys and secrets from environment variables or .env files. This way you never hardcode tokens in your scripts.

SILENCE read a variable from the system environment
ENV "GITHUB_TOKEN" INTO token
BEARER token INTO auth
SILENCE loads all keys from .env as CC variables
DOTENV ".env"
SPEAK api_key

A .env file looks like this — one key per line, no spaces around the =:

API_KEY=your_secret_key_here
BASE_URL=https://api.example.com
TOKEN=supersecrettoken

Keys load as lowercase CC variables. API_KEY becomes api_key.

⚠ Security Never share or commit your .env file. Add it to .gitignore so it stays off GitHub.
21

HTTP Requests

CC has four HTTP commands — one for each method. All require INVOKE HTTP at the top of your script. Every request automatically populates four response variables you can read immediately after the call.

Error Handling — OR FAIL / OR CONTINUE

Every request ends with either OR FAIL or OR CONTINUE. You must pick one.

OptionWhat happens when the request fails
OR FAILCC stops immediately and prints a clear error. Use this when failure means your script can't continue.
OR CONTINUECC keeps running. Check response_error yourself to decide what to do. Use this when you want to handle errors gracefully.

FETCH — GET request

Retrieve data from a URL. The most common HTTP operation.

INVOKE HTTP
FETCH "https://api.github.com" "/users/torvalds" INTO response OR FAIL
SPEAK response_status
SPEAK response_body

POST — send data

Send data to create a new resource. The body is a JSON string.

GRANT body TO "{\"name\": \"CC\", \"type\": \"language\"}"
POST "https://api.example.com" "/items" body INTO response OR FAIL

PUT — update existing data

GRANT body TO "{\"status\": \"active\"}"
PUT "https://api.example.com" "/items/1" body INTO response OR FAIL

DELETE — remove data

DELETE "https://api.example.com" "/items/1" INTO response OR CONTINUE

Response Variables

After every request, these four variables are automatically set:

VariableContainsExample
response_statusHTTP status code200, 404, 500
response_bodyThe full response text or JSON{"id": 1, "name": "..."}
response_error"none" on success, message on failure"404 NOT FOUND"
response_headersResponse headers as a stringContent-Type, Date, etc.
22

Auth Headers

CC has shortcuts for the most common authentication patterns. Build a header string, add it to a list, and pass the list name in your request.

Bearer Token — most common for modern APIs

DOTENV ".env"               SILENCE loads api_key from .env
BEARER api_key INTO auth
GATHER headers
ADD headers auth

FETCH "https://api.example.com" "/me" headers INTO response OR FAIL

Basic Auth — username and password

BASIC "username" "password" INTO auth
GATHER headers
ADD headers auth

Custom Header — any key/value pair

HEADER "X-Api-Version" "2" INTO ver
GATHER headers
ADD headers ver
💡 Multiple headers You can ADD as many headers as you need to the same list before making the request. They all get sent together.
23

JSON

Most APIs respond with JSON. CC has three commands for working with it — all require INVOKE JSON.

PARSE — turn a JSON string into something CC can work with

INVOKE JSON
FETCH "https://api.github.com" "/users/torvalds" INTO response OR FAIL
PARSE response_body INTO user

PLUCK — extract a single field by name

PLUCK user "login" INTO username
PLUCK user "public_repos" INTO repos
SPEAK username
SPEAK repos

PRETTY — print formatted, readable JSON

PRETTY user
SILENCE prints indented JSON instead of one long line
💡 Nested JSON If a PLUCK result is itself a JSON object or array, CC stores it as a JSON string. You can PARSE it again to go one level deeper.
24

INVOKE — Modules

Modules unlock additional features. Call INVOKE once at the top of your script before using any module commands.

ModuleWhat it unlocks
INVOKE HTTPFETCH, POST, PUT, DELETE requests
INVOKE JSONPARSE, PLUCK, PRETTY — JSON handling
INVOKE RANDOMGenerate random numbers
INVOKE MATHMath functions: sqrt, floor, ceil, abs, round, sin, cos, tan, log
INVOKE TIMEGet the current Unix timestamp

MODULE — RANDOM

Generate a random integer between two values (inclusive) and store it in a variable.

INVOKE RANDOM
RANDOM 1 100 INTO n
SPEAK n

MODULE — MATH

Apply a math function to a value and store the result.

INVOKE MATH
MATH sqrt 144 INTO r   SILENCE r = 12
MATH floor 3 INTO f   SILENCE f = 3
MATH abs 5 INTO a     SILENCE a = 5

Available functions: sqrt floor ceil abs round sin cos tan log

MODULE — TIME

Get the current Unix timestamp — the number of seconds since January 1, 1970. Useful for measuring how long requests take.

INVOKE TIME
TIME INTO start
FETCH "https://api.example.com" "/data" INTO response OR FAIL
TIME INTO end
SPEAK end - start

Operators

OpTypeDescription
+Math / StringAdd numbers, or concatenate strings
-MathSubtract
*MathMultiply
/MathDivide — returns a decimal if needed
MOD a b INTO rMathRemainder of a divided by b. e.g. MOD 10 3 INTO r → r = 1
> / <ComparisonGreater than / less than
>= / <=ComparisonGreater than or equal / less than or equal
==ComparisonEqual — works on both numbers and strings
!=ComparisonNot equal — works on both numbers and strings
⚠ No operator precedence CC evaluates left to right. 2 + 3 * 4 equals 20, not 14. If order matters, use intermediate variables: GRANT temp TO 3 * 4 then GRANT result TO 2 + temp.

Errors

When CC encounters a problem it stops and prints [CC] message. CC never fails silently — every error has a clear message telling you exactly what went wrong.

ErrorWhat caused it
IMMORTAL violationTrying to reassign a constant
Unknown variableUsing a variable name before it was GRANTed
Divide by zeroDividing any value by 0
Unknown contractCalling a CONTRACT that was never defined
Index out of boundsGET or REMOVE with an index that doesn't exist
File not foundREAD on a file that doesn't exist
Module not invokedUsing HTTP/JSON/RANDOM etc. without INVOKE first
HTTP error (OR FAIL)Non-2xx response when OR FAIL is used
Network error (OR FAIL)Timeout or unreachable host when OR FAIL is used
JSON parse errorPARSE on text that isn't valid JSON
Key not foundPLUCK with a key that doesn't exist in the data
Env var not foundENV with a variable that isn't set in the system
Infinite loopWHILE running more than 100,000 times
BETRAYManually triggered with the BETRAY command
PLAYGROUND
