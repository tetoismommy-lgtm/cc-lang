THE CC
LANGUAGE

Cold. Precise. Unforgiving. Code that commands.

Files: .cc Interpreted Action-First 3 Modules Lists + Files + Loops
01

Overview

CC is an interpreted programming language built around a simple rule: every line begins with a command. No ambiguity. No hesitation. You tell CC what to do, and it obeys.

Keywords are always UPPERCASE. Variable names are always lowercase. Strings live inside "double quotes". Everything else is a number.

SILENCE welcome to CC
IMMORTAL version TO 2
ASK name "who are you"
SPEAK "CC acknowledges you,"
SPEAK name
02

Running CC

Save your code in any file ending with .cc and run it from any terminal:

$ cc myprogram.cc

CC is on your PATH — this works from any folder on your machine. No Python visible. Just CC.

💡 Tip
During development, rebuild your executable after changes with:
pyinstaller --onefile --name cc interpreter.py
03

Syntax Rules

CC has four hard rules. Break them and she betrays you.

RuleDetail
KEYWORDS UPPERCASEAll built-in commands must be fully uppercase
variables lowercaseVariable and contract names must be lowercase
"Strings quoted"All text values must be in double quotes
One statement per lineEach command lives on its own line
⚠ Case sensitive
speak is not the same as SPEAK. CC will not recognize lowercase keywords.
04

SPEAK

Prints a value to the screen. Accepts strings, numbers, variables, or math expressions.

SPEAK "Hello world"
SPEAK x
SPEAK x + y
SPEAK 10 * 3 + 2
05

GRANT

Creates or updates a variable. Requires the keyword TO between the name and value. Can store numbers, strings, or expression results.

GRANT x TO 10
GRANT name TO "CC"
GRANT result TO x * 2 + 5
GRANT x TO x + 1
06

IMMORTAL

Creates a constant. Once set, it can never be changed. Attempting to reassign an IMMORTAL raises an error immediately.

IMMORTAL pi TO 3.14159
IMMORTAL max_lives TO 3
SPEAK pi
⚠ Error
GRANT pi TO 5 after IMMORTAL pi TO 3.14 → CC refuses — 'pi' is IMMORTAL.
07

ASK

Prompts the user for input and stores it in a variable. CC automatically detects whether the response is a number or string — no casting required.

ASK age "how old are you"
ASK name "what is your name"
SPEAK name
SPEAK age + 1
08

SILENCE

Comments. Everything after SILENCE on that line is ignored by CC. Must appear at the start of its own line.

SILENCE this is a comment
SILENCE --- setup phase ---
GRANT x TO 10
09

BETRAY

Throws a custom error and halts the program immediately. Use it to guard against impossible states.

ASK age "enter your age"
OBEY age < 0
    BETRAY "age cannot be negative"
END
SPEAK age
10

OBEY / ALSO OBEY / OTHERWISE

Conditional execution. Supports full else-if chains using ALSO OBEY and a final fallback with OTHERWISE. Every block ends with END.

Comparisons: > < >= <= == !=

ASK score "enter score"
OBEY score >= 90
    SPEAK "S rank"
ALSO OBEY score >= 70
    SPEAK "A rank"
ALSO OBEY score >= 50
    SPEAK "B rank"
OTHERWISE
    SPEAK "you have failed CC"
END
11

WHILE

Repeats a block as long as a condition remains true. CC will stop after 100,000 iterations to prevent infinite loops.

GRANT x TO 1
WHILE x <= 5
    SPEAK x
    GRANT x TO x + 1
END
12

REPEAT

Runs a block an exact number of times. The count can be a literal number or a variable.

REPEAT 3
    SPEAK "you have no choice"
END

GRANT n TO 5
REPEAT n
    SPEAK "obedience noted"
END
13

CONTRACT

Defines a named, reusable block of code. Call it by writing its name alone on a line. Contracts share memory with the program — all variables are accessible inside.

CONTRACT greet
    SPEAK "the contract is sealed"
    SPEAK "CC acknowledges you"
END

greet
greet
14

Lists

CC lists are ordered collections. Create one with GATHER, add items with ADD, retrieve by index with GET, and check the size with SIZE. Indexes start at 0.

GATHER scores
ADD scores 95
ADD scores 82
ADD scores 74

GET scores 0 INTO first
SIZE scores INTO total
REMOVE scores 1

SPEAK first
SPEAK total
CommandUsage
GATHER nameCreate an empty list
ADD name exprAppend a value to the list
GET name index INTO varRead a value by index into a variable
REMOVE name indexDelete item at index
SIZE name INTO varStore the list length into a variable
15

String Operations

CC provides three string operations. All results are stored INTO a variable.

GRANT first TO "Lelouch"
GRANT last TO "Lamperouge"

LENGTH first INTO n
SPEAK n

JOIN first last INTO full
SPEAK full

SLICE first 0 3 INTO part
SPEAK part

16

File I/O

CC can write to and read from files. WRITE creates or overwrites a file. READ loads a file's contents into a variable.

WRITE "notes.txt" "the contract is sealed"

READ "notes.txt" INTO contents
SPEAK contents
⚠ Note
WRITE overwrites without warning. READ on a missing file causes an error.
17

INVOKE — Module System

Modules must be invoked before use. Think of it as CC granting you access to additional powers. Available modules: RANDOM, MATH, TIME.

INVOKE RANDOM
INVOKE MATH
INVOKE TIME
⚠ Required
Using RANDOM, MATH, or TIME without invoking them first will cause an error.

MODULE — RANDOM

Generates a random integer between two values (inclusive) and stores the result.

INVOKE RANDOM
RANDOM 1 100 INTO n
SPEAK n

MODULE — MATH

Applies a math function to a value and stores the result.

INVOKE MATH
MATH sqrt 144 INTO r
MATH floor 3.7 INTO f
MATH abs -5 INTO a
SPEAK r
FunctionDescription
sqrtSquare root
floorRound down
ceilRound up
absAbsolute value
roundRound to nearest
sin / cos / tanTrig functions (radians)
logNatural logarithm

MODULE — TIME

Gets the current Unix timestamp (seconds since 1970) and stores it in a variable.

INVOKE TIME
TIME INTO t
SPEAK t

Errors

When CC encounters a problem she stops immediately. All errors print as [CC] message.

ErrorCause
IMMORTAL violationReassigning a constant
Unknown variableUsing a name before GRANT
Divide by zeroDividing any value by 0
Unknown contractCalling an undefined CONTRACT
Index out of boundsGET or REMOVE with invalid index
File not foundREAD on a missing file
Module not invokedUsing RANDOM/MATH/TIME without INVOKE
Infinite loopWHILE exceeding 100,000 iterations
BETRAYManually triggered in your code

GUESSING GAME
RANDOM + WHILE
SILENCE guess the number
INVOKE RANDOM
RANDOM 1 10 INTO secret
GRANT guess TO 0
WHILE guess != secret
    ASK guess "guess 1-10"
    OBEY guess < secret
        SPEAK "too low"
    ALSO OBEY guess > secret
        SPEAK "too high"
    END
END
SPEAK "correct"
